Web Utilities Index

Here is a collection of useful pages for examining the web. Some of these tools are useful for developing your web pages. Also included are some tools for XML and web services debugging. Please Enjoy!

  • OpenId Client Test - Test that your OpenId provider is working correctly with an independent client.
  • XML Grinder - Load XML in either by reading from a web URL, or by pasting from your text editor. This will parse the XML. It will allow you to clean up, reformat, and validate the XML using S-XML (Simple XML) rules.
  • sudoku solver - Fustrated by a particular SUDOKU puzzle? Enter the starting values here and have the puzzle solved. You can also play and make your own sudoku puzzle and see IF it can be solved.
  • page source viewer - This allows you to enter a URL and see exactly what the server sent. There is no interpretation of JavaScript, there is no attempt to parse XML (and fail), there is no processing of redirect commands. It simply reads that URL and display precisely what it receives. (old source viewer)
  • XML Stripper - Probably not what you are thinking. This will take a block of XML and strip all occurrances of a particular tag from it. This is useful in XPDL to remove all the ExtendedAttributes from an XPDL file.
  • page cleaner - Sometimes I am on a slow link, and the pages I am trying to browse through have an annoying amount of graphics and other large files attached. What I want to do is to prevent the downloading of all the junk I am not going to look at, and focus instead on the textual content which is what I am looking for. This is an attempt to make a utility which strips out all the graphics and leaves just the text. Scripts are disabled as well. In some cases links can still be followed, and in other cases the links (and maybe even the entire page) will be broken. This is actually not very useful in the current form, so I am looking for a better solution, but in some senses loading a page through the "cleaner" will be somewhat safer than loading it directly because it prevents a lot of the page side effects.
  • URL Hacker - paste a URL with a bunch of parameters and submit. This will parse the URL parameters out, and make a form with all the parameters, as well as input fields where you can change the parameter values. A button on that form allows you to access the original page, with the modified parameters. This is really useful when the original URL has large and complexly encoded parameter values.
  • Threads - produces a thread dump to show what tomcat is doing.
  • Unicode - enter text and see the unicode and UTF-8 equivalents.
  • LaTeX - enter text and see the LaTeX equivalents.
  • Decimal Formatter - experiment with decimal formatting in Java.
  • HTTPRequest Helper - demonstrating the various parts of the HTTP request path parsing.
  • Facebook Experiment - Facebook login and fetches.
  • Unbuilder - Look at output files and determine which source they came from.
